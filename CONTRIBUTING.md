# Contributing

Contributions are welcome.

1. Create a focused branch.
2. Do not add real Site IDs, deployment URLs, owner emails, bookmarks, credentials, tokens, cookies, logs, or local environment files.
3. Run `npm test`, `npm run lint`, and `npm run build`.
4. Describe user-visible changes and security implications in the pull request.

Use synthetic fixtures in tests. Follow `SECURITY.md` for vulnerability reports.
